Тест-кейс для микросервиса обьявлений

1. Создание обьявления (Post/ads)
ТС-1.1. Успешное создание обьявления
Предусловие: Уникальный seller_id в диапазоне 111111–999999
Шаги: 
1.Отправить POST запрос с валидным JSON
2.Проверить статус код 201
3.Проверить, что вернулся itemId
Ожидаемый результат: Обьявление создано, возвращается корректный ответ

ТС-1.2. Создание обьявления с неполными данными
Предусловие: Отсутствуют обязательные поля
Шаги:
1. Отправить POST запрос без обязательных полей
2. Проверить статус код 400
Ожидаемый результат: Возвращается ошибка валидации

ТС-1.3. Создание обьявления с неверным типом данных
Предусловие: Неправильные типы в полях
Шаги:
1. Отправить POST с srting вместо number в price
2.Проверить статус код 400
Ожидаемый результат: Возвращается ошибка валидации

2. Получение обьявления по ID(Get)

ТС-2.1. Успешное получение существующего документа
Предусловие: Обьявление создано
Шаги:
1. Отправить Get запрос с существующим itemId
2.Проверить статус код 200
3.Проверить соответсвие данным
Ожидаемый результат: Возвращаются корректные данные обьявления

ТС-2.2. Получение обьявления с несуществующим id
Предусловие: itemId не существует
Шаги:
1. Отправить Get запрос которого нет в базе itemId
2.Проверить статус код 404
Ожидаемый результат: Возвращаются 404 Not Found

3. Получение всех обьвлений продавца

ТС-3.1. Успешное получение обьявлений продавца
Предусловие: Продавец имеет обьявления
Шаги:
1. Отправить Get запрос с существующим itemId
2.Проверить статус код 200
3.Проверить соответсвие данным
Ожидаемый результат: Возвращаются корректные данные обьявления

ТС-3.2. Продавец без объявлений
Предусловие: Продавец существует, но без объявлений
Шаги:
1. Отправить GET запрос с sellerId без объявлений
2. Проверить статус код 200
3. Проверить пустой массив
Ожидаемый результат: Возвращается пустой массив []

ТС-3.3. Несуществующий продавец
Предусловие: sellerld не существует
Шаги:
1. Отправить GET запрос с несуществующим
sellerId
